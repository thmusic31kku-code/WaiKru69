# เว็บไซต์พิธีไหว้ครูดนตรี ภาคตะวันออกเฉียงเหนือ — มหาวิทยาลัยขอนแก่น

ระบบเว็บไซต์งานพิธีไหว้ครูดนตรี ประกอบด้วย 3 ระบบหลัก:
1. **ระบบบริจาคทำบุญ** — QR + เลขบัญชี, กรอกข้อมูลผู้บริจาค, แนบสลิป
2. **ระบบลงทะเบียน/สำรองที่นั่ง** — เข้าสู่ระบบด้วย Google, สำรองที่นั่งอัตโนมัติ, ออกใบยืนยัน PDF พร้อม QR
3. **ระบบแอดมิน** — แดชบอร์ด + CRUD ครบทุกส่วน, **เข้าสู่ระบบได้ 2 แบบ**: username/password
   (ไม่ต้องพึ่ง Google เลยก็ได้) หรือ Sign in with Google

> **ต้องการติดตั้งแบบทีละขั้นตอนโดยละเอียด (ตั้งแต่เขียน `Code.gs` ไปจนถึง `npm run dev`
> และนำขึ้น hosting)? ดูที่ [`INSTALL.md`](./INSTALL.md) — เอกสารนี้เป็นแค่ภาพรวมสรุปย่อ**

อ่านภาพรวมสถาปัตยกรรมและโครงสร้างฐานข้อมูลแบบละเอียดได้ที่ **`ARCHITECTURE.md`**

---

## สรุปขั้นตอนแบบย่อ

1. **Backend:** สร้าง Google Sheet ว่าง > Extensions > Apps Script > วางโค้ด `backend/Code.gs` >
   รัน `setupSpreadsheet` > ตั้งแอดมินคนแรก (`seedFirstAdminPassword` สำหรับ username/password
   หรือ `seedFirstAdmin` สำหรับ Google) > Deploy เป็น Web App > คัดลอก URL
2. **Config:** ใส่ `API_URL` (และ `GOOGLE_CLIENT_ID` ถ้าต้องใช้ Google) ใน `public/js/config.js`
3. **พัฒนาในเครื่อง:** `npm install` แล้ว `npm run dev` เปิดพรีวิวที่ `http://localhost:3000`
4. **Hosting:** ลากโฟลเดอร์ `public/` ขึ้น [app.netlify.com/drop](https://app.netlify.com/drop)
   หรือเชื่อม Git repo กับ Netlify (publish directory = `public`)

รายละเอียดทุกขั้นตอน คำเตือน และการแก้ปัญหาที่พบบ่อย อยู่ใน **`INSTALL.md`**

## การปรับแต่งที่ทำได้ทันทีโดยไม่ต้องแก้โค้ด

แก้ค่าตรง ๆ ในชีต **Config** แล้วรีเฟรชหน้าเว็บได้เลย เช่น เปลี่ยนราคาเงินกำนล, ชื่องาน,
วันที่, จำนวนที่นั่งทั้งหมด (ต้องรัน setup ใหม่ถ้าต้องการสร้างที่นั่งเพิ่มหลังจากมีข้อมูลจองแล้ว
ให้เพิ่มแถวในชีต `Seats` ด้วยมือแทน เพื่อไม่ให้กระทบข้อมูลเดิม)

## หมายเหตุด้านความปลอดภัยของเงินบริจาค

รูปภาพ QR ที่ใช้ในหน้า `donate.html` (`public/assets/donation-qr-scb.png`) คัดลอกมาจาก
โปสเตอร์ประชาสัมพันธ์งานจริงที่ผู้จัดงานเผยแพร่ ไม่ได้สร้าง QR ใหม่ขึ้นมาเอง เพื่อความถูกต้อง
ปลอดภัยของยอดโอนเงิน **หากมีการเปลี่ยนบัญชีธนาคาร** ให้ผู้ดูแลระบบนำภาพ QR ใหม่จากธนาคาร
(หรือแอปที่ออก QR ทางการ) มาแทนที่ไฟล์นี้โดยตรง และแก้ค่าบัญชีในชีต Config ให้ตรงกัน

## ข้อจำกัดที่ควรทราบ

- Apps Script Web App มีโควตาการเรียกใช้งานต่อวัน (ขึ้นกับประเภทบัญชี Google) หากงานมีผู้เข้าใช้
  พร้อมกันจำนวนมากในช่วงเวลาสั้น ๆ ควรพิจารณาย้ายไป Firebase/ฐานข้อมูลจริงในอนาคต
- การอัปโหลดสลิปใช้วิธีแปลงเป็น base64 แล้วส่งในตัว request เดียว เหมาะกับไฟล์รูปขนาดไม่เกิน
  ~5-8 MB หากรูปใหญ่กว่านี้ควรแนะนำผู้ใช้บีบอัดรูปก่อนอัปโหลด
- ระบบ session ของแอดมินอายุ 6 ชั่วโมง (ปรับได้ที่ `SESSION_TTL_SECONDS` ใน `Code.gs`)
